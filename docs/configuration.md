# Configuration

OMG uses Hydra configs under:

```text
configs/generation/
```

The training entry point is:

```text
configs/generation/train.yaml
```

## Top-Level Groups

```text
callbacks/       Lightning callbacks.
data/            Dataset and dataloader configs.
denoiser/        Transformer denoiser sizes.
diffusion/       Diffusion process config.
exp/             Experiment presets.
logger/          Logger presets.
loss/            Training loss config.
model/           Lightning module config.
paths/           Portable path defaults.
representation/ Motion representation and stats.
trainer/         Lightning trainer presets.
```

## Default Representation

```text
configs/generation/representation/125d.yaml
```

Important fields:

```yaml
feat_dim: 125
sequence_length: 60
num_prev_states: 10
stats_path: assets/stats/g1_125d_stats.json
```

`sequence_length=60` means each diffusion request generates a 60-frame future
chunk at the planner FPS.

`stats_path` points to the normalization file generated by
`omg.cli.generation.compute_stats`; it is not precomputed in this repository.

## Experiment Presets

Use `exp=<name>`:

```text
50m
100m
300m
500m
1b
```

Each preset sets model size, training scale, and output naming. Keep release
experiments in these presets; put one-off ablations in separate branches or
external config overrides.

## Trainer Presets

```text
trainer=1gpu
trainer=2gpu
trainer=4gpu
trainer=8gpu
```

Lightning DDP is configured through these presets. Per-process batch size lives
under:

```text
data.loader_opts.train.batch_size
data.loader_opts.val.batch_size
```

## Logger Presets

```text
logger=none
logger=wandb
```

Set W&B environment variables before using `logger=wandb`.

## Useful Overrides

```bash
trainer.max_steps=200000
trainer.val_check_interval=2000
callbacks.checkpoint.every_n_train_steps=2000
data.loader_opts.train.batch_size=64
ckpt_path=outputs/run/checkpoints/last.ckpt
```
